# Changelog

## v1.0.0 发布说明

> 发布日期：2026-05-20
> 里程碑：v1.0.0 — 基础功能完成

### 发布概述
v1.0.0 是掼蛋游戏项目的首个正式发布版本，实现了在线掼蛋对战平台的核心功能闭环，涵盖用户认证、房间管理、游戏对战和实时通信四大模块。

### 升级指南

#### 从 v0.x 升级到 v1.0.0

1. **数据库迁移**
   - 执行 `schema-v1.0.0.sql` 创建用户表、房间表和游戏记录表
   - 数据结构与 v0.x 不兼容，需重新初始化数据库

2. **后端升级**
   - JDK 要求：17+
   - Spring Boot 版本：3.1.5
   - Maven 依赖已全部更新，执行 `mvn clean install` 重新构建

3. **前端升级**
   - Node.js 要求：18+
   - 执行 `npm install` 更新依赖
   - 路由结构已调整，旧版书签可能失效

#### 向后兼容注意事项

- **API 端点变更**：所有接口路径已统一为 `/api/` 前缀，旧端点不再支持
- **Token 格式**：JWT Token 签名算法更新为 HS256，旧 Token 在升级后立即失效，用户需重新登录
- **WebSocket 协议**：消息格式已规范化，旧客户端无法与新服务端建立连接
- **数据库表结构**：用户表新增 `nickname`、`avatar_url` 字段，Room 表新增 `room_no` 6 位房间号字段

### v1.0.0 功能总览

| 模块 | 功能 | 状态 |
|------|------|------|
| 用户认证 | 注册、登录、退出、Token 刷新 | ✅ 已完成 |
| 房间管理 | 创建、加入、退出、解散、转移房主 | ✅ 已完成 |
| 游戏对战 | 准备、开始、出牌、回合管理 | ✅ 已完成 |
| 实时通信 | WebSocket 消息推送、状态同步 | ✅ 已完成 |
| 匹配服务 | 快速匹配、取消匹配、轮询结果 | ✅ 已完成 |

### 已知问题与限制

1. 游戏超时检测依赖定时任务轮询，实时性有限（30 秒间隔）
2. 断线重连后玩家手牌状态需前端重新请求同步
3. 快速匹配在低并发场景下等待时间较长
4. 暂无 AI 出牌建议功能（规划中）

### 优化

- 后端：Dockerfile 非 root 用户运行，安全加固
- 后端：Nginx 配置从 HTTP 升级为 HTTPS + HTTP/2
- 后端：Redis 服务增加健康检查
